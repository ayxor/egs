# Video Editor Service

Internal service for asynchronous media processing.

The Video Editor receives a resource through a URL, applies operations, and writes the result to another URL.  
The service is business-agnostic and authenticated via API key.

## API Contract

The official contract is documented in `video-editor.yaml` (OpenAPI 3.0.3).  
This README summarizes the same contract for quick reference.

Base URL (local): `http://localhost:8080`

Required header for all endpoints:

```http
X-API-Key: <key>
```

## Endpoints

| Method | Path | Description |
|---|---|---|
| POST | /jobs | Create a job with an ID generated by the service |
| PUT | /jobs/{job_id} | Create or replace a job with a fixed ID (idempotent) |
| DELETE | /jobs/{job_id} | Cancel a job |
| GET | /jobs/{job_id} | Get job state and progress |
| GET | /jobs/{job_id}/progress | SSE stream of job progress |
| GET | /jobs/{job_id}/operations | Operations requested for that job |
| GET | /jobs/operations | Operations supported by this instance |

## Request Model

### POST /jobs and PUT /jobs/{job_id}

```json
{
    "src_url": "https://storage/raw/abc123.mp4",
    "dst_url": "https://storage/processed/abc123.mp4",
    "progress_url": "https://composer/internal/jobs/progress",
    "operations": [
        {
            "type": "watermark",
            "params": {
                "text": "UA",
                "position": "bottom-right",
                "opacity": 0.8
            }
        }
    ]
}
```

Required fields:

- src_url  
- dst_url  
- operations (non-empty list)

## Job State

Response example for GET /jobs/{job_id}:

```json
{
    "job_id": "job_7f3a1b",
    "status": "processing",
    "percent": 45,
    "created_at": "2026-03-17T10:00:00Z",
    "updated_at": "2026-03-17T10:01:23Z"
}
```

Possible states:

- queued  
- processing  
- done  
- failed  
- cancelled  

## Expected HTTP Codes

- 200 OK: Successful read  
- 202 Accepted: Job accepted for asynchronous processing  
- 204 No Content: Cancellation accepted  
- 400 Bad Request: Invalid payload or missing fields  
- 401 Unauthorized: Invalid or missing API key  
- 404 Not Found: Job does not exist  
- 422 Unprocessable Entity: Unknown or unsupported operation  

## Design Notes

- The service manages processing jobs but not video file ownership.  
- Therefore, there is no `/videos` CRUD endpoint in this microservice.  
- `POST /jobs` creates a new job with an ID generated by the service.  
- `PUT /jobs/{job_id}` allows idempotent retries with a caller-controlled ID.  
- `DELETE /jobs/{job_id}` cancels jobs that are queued or processing.  

## How to Run

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 app.py
```

```bash
export VIDEO_EDITOR_API_KEY="my-key"
python3 app.py
```
