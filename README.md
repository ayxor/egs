# Video Editor Service

Internal service for asynchronous media processing.

The Video Editor receives a source file upload, applies operations, and writes the result to a destination URL.
The service is business-agnostic and authenticated via API key.

## API Contract

The official contract is documented in `video-editor.yaml` (OpenAPI 3.0.3).
This README summarizes the same contract for quick reference.

Base URL (local): `http://localhost:8080`

Required header for all endpoints:

```http
X-API-Key: <key>
```

## Endpoints

| Method | Path | Description |
|---|---|---|
| GET | /db/status | Check simple database status (SQLite) |
| POST | /jobs | Create a job with an ID generated by the service |
| PUT | /jobs/{job_id} | Create or replace a job with a fixed ID (idempotent) |
| DELETE | /jobs/{job_id} | Cancel a job |
| GET | /jobs/{job_id} | Get job state, progress, and requested operations |
| GET | /jobs/operations | Operations supported by this instance |

## Request Models

### POST /jobs (multipart/form-data)

Required form fields:

- file (binary)
- dst_url
- operations (JSON string with a non-empty operations array)

Optional form fields:

- progress_url

Example:

```bash
curl -X POST "http://localhost:8080/jobs" \
  -H "X-API-Key: stub-api-key" \
  -F "file=@./sample.mp4" \
  -F "dst_url=https://storage/processed/abc123.mp4" \
  -F 'operations=[{"type":"watermark","params":{"text":"UA","position":"bottom-right","opacity":0.8}}]'
```

### PUT /jobs/{job_id} (application/json)

```json
{
  "src_url": "https://storage/raw/abc123.mp4",
  "dst_url": "https://storage/processed/abc123.mp4",
  "progress_url": "https://composer/internal/jobs/progress",
  "operations": [
    {
      "type": "watermark",
      "params": {
        "text": "UA",
        "position": "bottom-right",
        "opacity": 0.8
      }
    }
  ]
}
```

Required JSON fields:

- src_url
- dst_url
- operations (non-empty list)

Optional JSON field:

- progress_url

## Job State

Response example for GET /jobs/{job_id}:

```json
{
  "job_id": "job_7f3a1b",
  "status": "processing",
  "percent": 45,
  "progress": {
    "percent": 45
  },
  "operations": [
    {
      "type": "watermark",
      "params": {
        "text": "UA",
        "position": "bottom-right",
        "opacity": 0.8
      }
    }
  ],
  "source_ref": "upload://sample.mp4",
  "source_file_name": "sample.mp4",
  "source_file_mime_type": "video/mp4",
  "created_at": "2026-03-17T10:00:00Z",
  "updated_at": "2026-03-17T10:01:23Z"
}
```

Possible states:

- queued
- processing
- done
- failed
- cancelled

## Expected HTTP Codes

| HTTP Code | Meaning | Typical cases in this API |
|---|---|---|
| 200 OK | Successful read | `GET /db/status`, `GET /jobs/{job_id}`, `GET /jobs/operations` |
| 202 Accepted | Accepted for async processing | `POST /jobs`, `PUT /jobs/{job_id}` |
| 204 No Content | Action accepted without body | `DELETE /jobs/{job_id}` |
| 400 Bad Request | Invalid request payload | Missing `file`, `dst_url`, or `operations` in `POST /jobs` |
| 401 Unauthorized | Missing or invalid API key | Any endpoint without correct `X-API-Key` |
| 404 Not Found | Resource does not exist | Unknown `job_id` |
| 422 Unprocessable Entity | Unsupported operation | `operations` contains unknown `type` |
| 500 Internal Server Error | Server-side failure | SQLite/database unavailable |

## Design Notes

- `POST /jobs` receives the source file directly and always generates a `job_id`.
- `GET /jobs/{job_id}` now includes both `operations` and `progress`.
- There is no `/resources/{resource_id}/state` endpoint.
- The service includes a simple SQLite database (`video_editor.db` by default).
- `GET /db/status` is used to verify database availability quickly.
- The service manages processing jobs but not file ownership.

## How to Run

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python3 app.py
```

Custom API key (optional):

```bash
export VIDEO_EDITOR_API_KEY="my-key"
python3 app.py
```

Custom DB file path (optional):

```bash
export VIDEO_EDITOR_DB_PATH="/tmp/video_editor.db"
python3 app.py
```

## Basic Frontend Demo

This service includes a small frontend page served by Flask at:

```text
http://localhost:8080/
```

The page is meant only as a quick proof that the API works. It lets you:

- set `X-API-Key`
- load supported operations (`GET /jobs/operations`)
- create a job (`POST /jobs`, multipart upload)
- watch progress (`GET /jobs/{job_id}` polling)
- cancel the current job (`DELETE /jobs/{job_id}`)
- check database status (`GET /db/status`)

Use the same API key configured in the backend (`VIDEO_EDITOR_API_KEY`, default `stub-api-key`).
